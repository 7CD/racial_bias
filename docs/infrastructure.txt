1. В качестве веб-сервера HTTP используем nginx
    1.1. nginx слушает 80 порт и проксирует запросы между сервисами.
         Возможно масштабировать приложение, перенаправлять разные запросы в разные сервисы.

2. В качестве сервера приложений используем uWSGI
    2.2. uWSGI общается с nginx через сокет :8080

3. Структура приложения:
    flask_app 
    ├── docker-compose.yml          <-- Инструкции сборки приложения. Комментарии внутри.
    ├── docs                        <-- Документация
    │   └── infrastructure.txt      <-- Описание инфраструктуры
    ├── flask                       <-- Директория сервиса Flask
    │   ├── Dockerfile              <-- Инструкции для сборки приложения flask. Комментарии внутри
    │   ├── app                     <-- Исполнительная директория приложения Flask
    │   │   ├── __init__.py         <-- файл, в коором мы импортируем Flask и создаем объект app, который будет вызывать uWSGI. Импортируем маршрутизаторы
    │   │   ├── static              <-- Фронтенд папка
    │   │   │   ├── css             
    │   │   │   │   └── style.css   <-- Стили
    │   │   │   └── js
    │   │   │       └── app.js      <-- кастомные JS
    │   │   ├── templates           <-- Папка для всех шаблонов, используется в render_template
    │   │   │   ├── base.html       <-- Дефолтный шаблон для все страниц приложения, который содержит шапку, подвал и ссылки на вызываемый контент
    │   │   │   └── index.html      <-- Стартовая страница
    │   │   └── views.py            <-- Маршрутизации
    │   ├── app.ini                 <-- uWSGI конфигурации. Описание внутри
    │   ├── requirements.txt        <-- инструкции pip
    │   └── run.py                  <-- файл, который будет вызывать объект приложения при запуске uWSGI
    ├── log                         
    │   ├── nginx 
    │   └── uwsgi
    └── nginx
        ├── Dockerfile              <-- Заменяем дефолтный файл с конфигами и подкидываем nginx свои конфиги из nginx.conf 
        └── nginx.conf              <-- Содержит набор конфигураций nginx, которомы мы подменяем стандартные конфиги
